# ISA FIRE Tracker

Personal finance web app for tracking UK Stocks & Shares ISAs, SIPPs, and GIAs across multiple providers, with a FIRE calculator (projections, Monte Carlo, drawdown). Supabase handles auth and cloud persistence with a localStorage cache; single admin user (`VITE_ADMIN_EMAIL`). Hosted on Vercel.

A React Native port lives in the sibling folder `../isa-fire-mobile` — changes here do not automatically apply there.

## Stack

- **React 19 + Vite 8** — TypeScript, Tailwind CSS v4
- **Supabase** — auth + cloud persistence (`lib/supabase.ts`, `lib/db.ts`, `lib/localCache.ts`)
- **Recharts** — charts
- **xlsx** — Vanguard fund-holdings uploads for the Look-Through tab
- **vite-plugin-pwa** — installable PWA
- **Vitest** — tests for all money maths (`src/lib/*.test.ts`, `store.test.ts`)

## Commands

```bash
npm run dev      # Vite dev server
npm run build    # tsc -b && vite build — must pass before any commit
npm test         # vitest run
npm run lint
```

## Workflow rules

- **Always run `npm run build` before committing.** Vercel fails the deploy on TypeScript errors that `npm run dev` tolerates — do not push until the build is clean.
- Run `npm test` after touching anything in `src/lib/` or `src/store.ts` — the financial calculations are all covered by Vitest.
- `PLAN-*.md` files in the repo root are implementation plans (generated by the `/plans` command). When you finish executing one, delete it.

## Environment

`.env.local` (also set in Vercel): `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`, `VITE_ADMIN_EMAIL`.

Supabase schema is in `supabase-schema.sql`; SQL changes are applied via the Supabase MCP server (or by hand in the dashboard SQL editor if the MCP is unavailable).

## Conventions

- Currency: base is GBP; conversions go through `CurrencyContext` + `fxRates.ts`. Never convert already-converted values (this caused real bugs — round-trip tests exist in `currencyRoundTrip.test.ts`).
- All new money maths must come with a Vitest test alongside it in `src/lib/`.
